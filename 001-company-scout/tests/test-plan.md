# 테스트 플랜 (= 보고서 합격 자가점검)

이 도구의 산출물은 코드가 아니라 **SKILL.md가 만드는 보고서**다. 따라서 "테스트"는 보고서가 합격 기준을 만족하는지 점검하는 체크리스트로 운영한다. 테스트하지 않은 것을 테스트했다고 말하지 않는다.

## 테스트 범위
- 포함: 파일럿 회사(마이리얼트립) 보고서가 출력 스키마·출처 규칙·초심자 기준을 만족하는지.
- 제외: 자동 단위테스트(프롬프트 기반 스킬이라 해당 없음), 실데이터 API 연동(공개/더미로 대체).

## 합격 체크리스트 (보고서 1건당)
| # | 항목 | 통과 기준 | 상태(마이리얼트립 골든예시) |
|---|---|---|---|
| 1 | 요약 먼저 | 맨 앞에 회사 한 줄 + 핵심 숫자 + 알 수 있는 것/없는 것 | ✅ |
| 2 | 출처 강제 | 사실 주장 100% 공개 URL+등급, 없으면 "근거 부족"/"가설" | ✅ (출처 33, A9·B18·C6) |
| 3 | 문제 후보 | ≥3건, 각 출처 + Codex 적합성 3축 점수 | ✅ (4건) |
| 4 | 사실/가설 분리 | [사실]과 [가설] 라벨 분리 + "분석의 경계" 섹션 | ✅ |
| 5 | 재무 신호 | A급(DART 등) 가능 시 추세·비율, 불가 시 근거 부족 명시 | ✅ (확정/근거부족 경계 명시) |
| 6 | 꼬리질문 | 각 섹션 끝 3개 | ✅ |
| 7 | 용어 설명 | 전문용어 전부 정의(또는 인라인) | ✅ |
| 8 | 초심자 가독성 | 비전문가가 읽고 회사·문제 설명 가능 | ✅ (영업이익↔순이익 설명 포함) |
| 9 | 본선 문제 회피 | 유튜브 공개 본선 문제는 인접 변형만 | ✅ |
| 10 | 포맷 정합성 | Codex `plugin.json` 규격 / Claude `.claude/skills/` 동작 | ✅ |

## 예외 케이스 (보고서가 다뤄야 함)
- 공개 출처 빈약 회사 → 해당 축 "근거 부족"으로 정직 표기(지어내지 않음).
- 출처 충돌(예: 영업이익) → 양쪽 병기 + "재확인 필요".
- C단서만 있는 항목 → 사실 단정 금지.

## 통과 / 실패 기준
- 통과: 위 10개 전부 충족.
- 실패(중단): 출처 없는 주장이 사실로 섞임 / 미설명 전문용어로 초심자가 막힘 / 추정을 사실로 단정.

## 미검증 (정직 표기)
