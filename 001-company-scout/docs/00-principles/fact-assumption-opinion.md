# 사실 / 가정 / 의견 / 추천 분리 가이드

제0지침 10번을 실행하기 위한 라벨 규칙. 모든 분석 문서는 주장을 아래 4종으로 구분한다.

## 라벨 정의

| 라벨 | 의미 | 표기 | 요건 |
|---|---|---|---|
| 사실 (FACT) | 출처로 검증 가능한 것 | `[사실]` | 레퍼런스 번호 필수 |
| 가정 (ASSUMPTION) | 검증 안 됐지만 진행 위해 둔 전제 | `[가정]` | 검증 방법·리스크 명시 |
| 의견 (OPINION) | 판단·해석 | `[의견]` | 근거 함께 |
| 추천 (RECOMMENDATION) | 다음 행동 제안 | `[추천]` | 사실/가정 기반임을 밝힘 |

## 작성 예시

- `[사실]` Next.js App Router는 서버 컴포넌트를 기본값으로 한다. [1]
- `[가정]` 초기 사용자는 월 1,000명 미만일 것이다. (검증: 출시 후 4주 트래픽 측정)
- `[의견]` 이 규모에서는 관리형 DB가 운영 부담을 줄인다.
- `[추천]` MVP는 단일 리전 + 관리형 Postgres로 시작한다.

## 체크리스트

- [ ] 모든 사실에 레퍼런스가 붙었는가
- [ ] 가정에 검증 방법이 있는가
- [ ] 의견과 사실이 섞이지 않았는가
- [ ] 추천이 무엇에 근거하는지 밝혔는가
