# 문제 정의

Gate 1 필수 산출물.

## 문제 한 문장
> **공모전 참가자(나)**가 AX 인재전쟁 2026 예선 Codex 플러그인을 기획할 때, **출제 기업의 "공개·검증 가능한 실제 문제"를 출처와 함께 빠르고 정확하게 파악하지 못하는** 문제를 겪는다. (예선은 공개 자료만 근거로 인정하므로, 부정확하거나 출처 없는 문제 정의는 곧 채점 탈락이다.) [1]

## 현재 어떻게 해결하나 (대안)
| 대안 | 한계 |
|---|---|
| 수동 웹서핑 + 메모 | 출처 누락·검증 안 됨·재현 불가. 7개사 반복 시 시간 폭증 |
| 일반 LLM에게 "이 회사 문제 알려줘" | 출처 없는 환각 위험. 예선 채점에서 근거 불인정 [1] |
| 유튜브 시리즈 시청 | clue일 뿐. 고정댓글 "공개되는 문제는 본선 문제, 예선은 참고만" [2] |

## 문제의 크기 / 빈도
- 영향 받는 사용자: 1명(본인). `[사실]` 개인용 도구.
- 발생 빈도: 출제 7개사 × 문제 탐색 반복. 파일럿은 1개사(마이리얼트립). `[가정]` 1개사당 검증된 문제 후보 3~5건이면 기획 입력으로 충분.

## 성공하면 무엇이 달라지나
- 출처가 달린 문제 후보 리스트가 생겨, Codex 플러그인 기획을 **검증 가능한 근거 위에서** 시작할 수 있다.
- 예선 제출 5문항(무엇을·왜·작동방식·AI활용·검증) 중 "왜 이 문제" "어떻게 검증"에 바로 답할 출처가 확보된다. [1]

## 사실 / 가정 / 의견
- `[사실]` 예선 제출물 = Codex 플러그인, 출처는 공개·AI 확인 가능 자료만 인정, 내부 비공개·출처 없는 숫자는 불인정. [1]
- `[사실]` 제출 마감 2026-07-10 23:59:59. [1]
- `[가정]` 파일럿 1개사 끝까지 검증 → 실효성 확인되면 나머지 확장. (사용자 지정 전략)
- `[의견]` 파일럿은 마이리얼트립이 적합 — 유튜브 2화로 제품/문제 맥락 단서가 이미 있어 출처 탐색 시작점이 명확. [2]
