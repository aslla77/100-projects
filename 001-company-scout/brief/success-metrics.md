# 성공 지표

Gate 1 필수 산출물. (2026-06-28 북극성 재정의)

## 북극성 (1순위)
> **초심자도 이 보고서 하나로 그 회사를 이해할 수 있는가.**
> 회사가 뭐 하는 곳인지 → 어떻게 돈 버는지 → 시장·경쟁 맥락 → 무슨 문제를 겪는지를, 전문지식 없이도 따라 읽을 수 있어야 한다. (공모전/Codex 포맷 부합은 2순위 제약일 뿐 목적이 아니다.)

| KPI(북극성) | 목표 | 측정 방법 |
|---|---|---|
| 초심자 이해도 | "이 회사가 뭐고 왜 그 문제를 겪는지" 보고서만 읽고 설명 가능 | 자가체크: 전문용어 전부 용어설명에 있음 / 논리 흐름 끊김 없음 / 요약 먼저 / 사실·가설 구분 명확 |

## 핵심 KPI (2순위 — 신뢰성·포맷)
| KPI | 목표값 | 측정 방법 |
|---|---|---|
| 출처 검증율 | 100% (사실 주장에 공개 URL, 없으면 "근거 부족") | 주장-출처 매핑 + 출처-주장 일치 점검 |
| 문제(이슈) 후보 | ≥ 3건, 각 [사실]/[가설] 라벨 + 출처 + Codex 적합성 루브릭 점수 | 보고서 문제 섹션 |
| 사실 / 가설 분리 | 확정 사실과 이슈 예측(가설)을 시각적으로 분리 | "분석의 경계" 섹션 존재 |
| 예선 포맷 정합성 | `src/.codex-plugin/plugin.json` + Codex 실행 가능 | 구조 점검 |

## 보조 지표
| 지표 | 목표 | 측정 |
|---|---|---|
| 출처 등급 | A(공식·DART 감사보고서·표준) 우선, C는 단서 표기 | source-log 등급 |
| 재무 신호 | 가능하면 DART 감사보고서(A급)로 수익성·비용·재고 추세 | 재무 섹션(없으면 근거부족) |
| 꼬리질문 | 각 섹션 끝 3개 | 보고서 |

## 실패 신호 (중단/재검토)
- 출처 없는 주장이 사실로 섞임 → 즉시 중단.
- 초심자가 읽다 막히는 미설명 전문용어 잔존 → 용어설명 보강.
- 확정 못 할 추정을 사실처럼 단정 → 가설로 강등.

## 출시 기준 (Launch criteria)
- [ ] 파일럿(마이리얼트립) 보고서를 비전문가가 읽고 회사·문제를 설명 가능(자가체크 통과).
- [ ] 사실 주장 100% 공개 출처 / 미확인은 "근거 부족" 또는 "가설".
- [ ] 문제 후보 ≥3건(사실·가설 분리, 출처, 적합성 점수).
- [ ] 보고서에 "분석의 경계(말할 수 있는 것/없는 것)" + 용어 설명 + 섹션별 꼬리질문 3개.
- [ ] Codex 플러그인 포맷 부합.

## 사실 / 가정
- `[사실]` 예선 출처 인정 기준 = 공개·AI 검증 가능 자료 [1].
- `[가정]` "초심자 이해"는 정성 기준 → 자가체크 리스트로 운영. 외부 사용자 테스트는 범위 밖(개인 도구).
