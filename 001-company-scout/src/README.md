# company-scout (Codex 플러그인)

회사명 1개를 입력하면 **공개 자료만으로** 그 회사를 `회사 → 제품 → 고객 → 재무신호 → 문제 → 관련회사`로 구조화하고, **초심자도 이해할 수 있는 출처 검증 보고서**를 만든다.

## 구조 (OpenAI Codex 플러그인 규격)
```
src/
├── .codex-plugin/
│   └── plugin.json              # 필수 매니페스트 (name/version/description/skills)
├── skills/
│   └── company-scout/
│       └── SKILL.md             # 분석 방법론·출처 규칙·출력 스키마
├── examples/
│   └── myrealtrip-dossier.md    # 골든 예시(few-shot)
└── README.md
```
규격 출처: OpenAI Codex — Build plugins (https://developers.openai.com/codex/plugins/build, 확인일 2026-06-28).

## 사용
Codex에서 플러그인 설치 후 스킬을 호출하고 회사명을 입력한다. 결과 보고서는 `out/<회사명>-dossier-YYYYMMDD.md` 로 저장한다.

## 포지셔닝
이 플러그인은 AX 인재전쟁 예선 **제출물 자체가 아니라**, 출제 기업의 문제를 푸는 제출용 플러그인을 기획하기 위한 **입력(회사 이해·문제 발굴) 도구 + 제출 포맷 연습**이다.

---

## 예선 제출 5문항 초안 (이 플러그인 기준)

1. **무엇을·누가·어떤 상황에서 쓰나**
   공모전 참가자(또는 회사를 빠르게 파악해야 하는 사람)가, 잘 모르는 회사 1곳을 공개 자료만으로 이해해야 할 때 쓴다. 회사명을 넣으면 출처가 검증된 기업 이해 보고서가 나온다.

2. **왜 이 문제를 선택했나**
   AX 인재전쟁 예선은 "공개·검증 가능한 자료"만 근거로 인정한다. 출제 기업의 실제 문제를 출처와 함께 정확히 파악하지 못하면 그 위에 세운 기획이 통째로 무너진다. 그 앞단(회사 이해·문제 발굴)을 빠르고 검증 가능하게 만드는 것이 이 도구의 문제다.

3. **플러그인은 어떻게 작동하나**
   SKILL.md가 회사 유형 판별 → 5축 공개 출처 수집 → 출처 등급(A/B/C)·일치 점검 → 문제 후보 추출(Codex 적합성 루브릭) → 초심자용 보고서 조립을 지시한다. 사실과 가설을 분리하고, 확인 못 한 것은 "근거 부족"으로 남긴다.

4. **AI를 어떻게 활용했나**
   (build 과정에서 실제로 사용한 AI 도구·방식을 여기에 기록 — logs/ 의 원본 대화 로그와 일치해야 함. 사후 편집 금지.)

5. **어떻게 검증했나**
   파일럿 회사(마이리얼트립)로 end-to-end 1회 실행해, 합격 체크리스트(문제 후보 ≥3·사실 주장 100% 출처·사실/가설 분리·분석의 경계·꼬리질문·용어 설명·초심자 가독성)를 통과하는지 자가 점검했다. 골든 예시는 `examples/myrealtrip-dossier.md`.

> ⚠️ 4·5번의 구체 내용은 실제 제출 시점에 실제 로그·검증 결과로 채운다(지어내지 않음).
