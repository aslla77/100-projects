# sync-docs

## 목적
구현과 문서의 불일치를 찾아 최신화한다 (Gate 6 직전).

## 읽어야 할 파일
- README.md, CLAUDE.md, planning/*, adr/*, docs/**, release/*, ops/*
- 최근 변경 코드, 테스트, 의존성 파일, 환경변수 예시

## 업데이트할 파일
- 불일치가 발견된 모든 문서 (README, planning, ADR, docs, release 등)

## 절차
1. 변경된 코드와 테스트를 확인한다.
2. README의 실행법·환경변수·기술 스택·현재 상태가 최신인지 확인한다.
3. planning 문서와 실제 구현이 충돌하지 않는지 확인한다.
4. ADR이 필요한 결정이 있었는지 확인한다.
5. API·데이터 모델·배포 방식·명령어 변경 여부를 확인한다.
6. 변경이 있으면 관련 docs를 업데이트한다.
7. 기술 설명에는 공식 문서 레퍼런스를 붙인다.

## 출력 형식
- 업데이트한 문서 목록, 발견한 불일치, 수정 내용, 남은 문서 부채, 릴리즈 전 필수 수정

## 멈춰야 하는 지점
- 코드와 문서가 충돌하는데 정답이 불확실하면 추측하지 말고 질문한다.

## 레퍼런스 규칙
`.claude/rules/documentation.md` 준수.
