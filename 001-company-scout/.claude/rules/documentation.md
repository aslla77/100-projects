# Rule: 문서화

- 동작이 바뀌면 README와 관련 문서를 같은 변경에서 업데이트한다.
- 구현과 planning 문서가 어긋나지 않게 유지한다 (sync-docs).
- 중요한 기술 결정은 ADR로 기록한다.
- 분석성 문서 끝에는 References 섹션을 둔다.
- CLAUDE.md는 짧게 유지하고, 긴 절차는 skills, 경로 규칙은 rules, 긴 설명은 docs로 분리한다.
- 빈 파일을 만들지 않는다. 각 문서는 무엇을 적는지 안내해야 한다.
