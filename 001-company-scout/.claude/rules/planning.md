# Rule: 기획

- 먼저 가장 작은 MVP를 정의한다. 포함/제외를 명확히 나눈다 (`feature-scope.md`).
- 범위 폭주를 막는다. 과설계 금지. 위험도 등급에 맞춘다.
- 중요한 기술 결정은 ADR(`adr/000N-*.md`)로 기록한다 — 선택지·트레이드오프·출처 포함.
- 사실·가정·의견·추천을 구분한다. 가정에는 검증 방법을 적는다.
- 코딩 전 plan-redteam으로 계획을 공격하고, planner가 한 번 수정한다.
- planning → build 전환 전 사람 승인을 요청한다.
