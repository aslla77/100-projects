# Rule: 코딩

- 승인된 범위만 구현한다. 작은 단위로 작업한다.
- 기능 하나마다 관련 테스트를 추가한다.
- 기존 구조를 무너뜨리지 않는다. 사용하지 않는 코드를 남기지 않는다.
- 비밀키·토큰·API 키를 코드에 직접 넣지 않는다 (환경변수/시크릿 매니저).
- 외부 라이브러리 추가 시 이유를 설명한다.
- 범위 변경은 먼저 planning 문서를 수정하고 이유를 기록한 뒤 진행한다.
- 동작이 바뀌면 README/관련 문서를 업데이트한다.
- 주석은 의도가 non-obvious할 때만 1줄.
- 에러 처리·검증은 시스템 경계(외부 API·유저 입력)에 둔다.
- 대규모 코드 변경·마이그레이션은 사람 승인 후 진행한다.
