# PRD (Lite)

Gate 3 필수. 가볍게 유지(Tier 0, 과설계 금지). 산출물 = **Codex 플러그인 `company-scout`**.

## 목표
> 회사명 1개를 입력하면, **공개 자료만으로** 그 회사를 `회사→제품→고객→문제→관련회사`로 "싹싹 털어" 구조화하고, 모든 주장에 공개 출처(등급 A/B/C)를 단 **company dossier(마크다운)** 와 **Codex 플러그인으로 풀 만한 문제 후보(출처+적합성 메모)** 를 산출하는 Codex 스킬을 만든다.

## 타깃 사용자
- 나(공모전 참가자). AX 인재전쟁 예선 Codex 플러그인 기획의 입력으로 사용. 동시에 예선 제출 포맷 연습.

## 핵심 사용자 스토리
- As a 참가자, I want 회사명만 넣으면 공개 출처 기반 회사 분석 dossier가 나오게, so that 출처 검증된 문제 후보로 바로 기획에 들어간다.
- As a 참가자, I want 모든 문제 후보에 공개 URL 출처와 A/B/C 등급이 강제로 붙게, so that 예선 채점("공개·검증 가능 자료만")을 통과한다.
- As a 참가자, I want 출처 못 댄 항목이 "근거 부족"으로 분리 표기되게, so that 환각을 제출물에 넣지 않는다.
- As a 참가자, I want 관련 회사와 관계 유형(투자사·경쟁사·고객사 등)도 함께 나오게, so that 회사 맥락을 빠짐없이 파악한다.

## 보고서 출력 스키마 (고정)
> 메인 축은 우리 것(`회사→제품→고객→문제→관련회사`). 참고 보고서(메디테라피 19장)는 *단서/예시*일 뿐 메인 축·수치 차용 금지.
0. **요약/표지** — 회사 한 줄 + 핵심 숫자 스냅샷 + "이 보고서로 알 수 있는 것/없는 것" 한 줄
1. **회사** — 뭐 하는 곳인가(초심자 눈높이), 연혁·규모·사업모델
2. **제품** — 무엇을 파나
3. **고객** — 누가 왜 쓰나
4. **재무 신호** — (DART 감사보고서 A급 가능 시) 매출·수익성·비용·재고 추세, 사실만
5. **시장·경쟁/관련 회사** — 업계 맥락 + 관련사(관계유형) + 경쟁사 수치
6. **문제 신호 → 이슈 가설** — [사실]/[가설] 분리, 문제 후보 ≥3(출처·적합성 점수)
7. **분석의 경계** — 공개자료로 말할 수 있는 것 / 없는 것
- (각 섹션 끝) **꼬리질문 3개**
- **References**(A/B/C·확인일) · **용어 설명** · **근거 부족 모음**

## 범위 제외 (이번에 안 함)
- 유료/비공개 API 연동(항공 가격 등 실데이터) — 공개 출처·더미로 한정.
- 특정 회사 문제를 **실제로 푸는** 플러그인(예: 럭키글라이드형 가격탐색) — 그건 별도 예선 제출물. 본 프로젝트는 그 입력을 만드는 '스카우트' 도구.
- 다중 회사 배치 처리, 웹 UI, 자동 스케줄링.
- 회사 관계 그래프 시각화(표까지만).
